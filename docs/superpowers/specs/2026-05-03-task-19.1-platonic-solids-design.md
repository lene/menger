# Design: Task 19.1 — Platonic Solids (Tetrahedron, Octahedron, Dodecahedron, Icosahedron)

**Date:** 2026-05-03
**Sprint:** 19
**Status:** Approved

---

## Summary

Add tetrahedron, octahedron, dodecahedron, and icosahedron as first-class triangle-mesh primitives
alongside the existing `cube`. Each is accessible via `--objects type=<name>` on the CLI and renders
through the existing `TriangleMeshSceneBuilder` path without engine changes.

---

## Architecture

### New files (one per solid)

```
menger-app/src/main/scala/menger/objects/Tetrahedron.scala
menger-app/src/main/scala/menger/objects/Octahedron.scala
menger-app/src/main/scala/menger/objects/Dodecahedron.scala
menger-app/src/main/scala/menger/objects/Icosahedron.scala
```

Each is a `case class` that `extends Geometry with TriangleMeshSource`, mirroring `Cube.scala`
exactly. Constructor: `(center: Vector3, scale: Float, material: Material, primitiveType: Int)`.
Only `toTriangleMesh: TriangleMeshData` is needed by the OptiX pipeline; `getModel` can be a
stub (throw `UnsupportedOperationException`) since these solids are not used in the LibGDX
preview path.

### Modified files

## Vertex Data

All solids are centered at the origin and **inscribed in a unit sphere** (all vertices at distance 1).
The `scale` field uniformly scales after centering; `center` translates.

### Tetrahedron — 4 vertices, 4 triangular faces (12 indices)

Standard symmetric form. No irrational constants required beyond `1/sqrt(2)` or equivalent floats.

Vertices (approximate, normalized to unit sphere):
```
( 0.0,      0.0,      1.0     )
( 0.9428f,  0.0,     -0.3333f )
(-0.4714f,  0.8165f, -0.3333f )
(-0.4714f, -0.8165f, -0.3333f )
```

### Octahedron — 6 vertices, 8 triangular faces (24 indices)

Axis-aligned, trivially on unit sphere:
```
(±1, 0, 0), (0, ±1, 0), (0, 0, ±1)
```

### Icosahedron — 12 vertices, 20 triangular faces (60 indices)

Uses golden ratio `φ = (1 + √5) / 2`, defined in companion object:
```scala
val Phi: Float = ((1.0 + math.sqrt(5.0)) / 2.0).toFloat
```
Vertices are all permutations of `(0, ±1, ±φ)`, normalized to unit sphere by dividing by
`sqrt(1 + φ²)`.

### Dodecahedron — 20 vertices, 12 pentagonal faces

Also uses `φ` (dual of icosahedron). Vertex sets:
- `(±1, ±1, ±1)` — 8 vertices
- Permutations of `(0, ±1/φ, ±φ)` — 12 vertices

Pentagonal faces fan-triangulated from face centroid → **3 triangles per pentagon = 36 triangles**
(108 indices).

All faces wound **CCW when viewed from outside**. Each solid uses the natural orientation
arising from its vertex formula (no forced uniform orientation).

## `ObjectType` Changes

### New constant

```scala
val TRIANGLE_MESH_TYPES: Set[String] =
  Set("cube", "parametric", "tetrahedron", "octahedron", "dodecahedron", "icosahedron") ++
  SPONGE_TYPES.filter(_ != "cube-sponge") ++
  PROJECTED_4D_TYPES
```

### Rewritten `isTriangleMesh`

```scala
def isTriangleMesh(objectType: String): Boolean =
  TRIANGLE_MESH_TYPES.contains(normalize(objectType))
```

### Updated `VALID_TYPES`

```scala
val VALID_TYPES: Set[String] = Set(
  "sphere", "cube", "parametric",
  "tetrahedron", "octahedron", "dodecahedron", "icosahedron",
  "sponge-volume", "sponge-surface", "cube-sponge", "sponge-recursive-ias",
  "tesseract", "tesseract-sponge-volume", "tesseract-sponge-surface"
)
```

## MeshFactory Changes

Four new case branches added alongside the existing `"cube"` branch:

```scala
case "tetrahedron"  => Tetrahedron(center, spec.size).toTriangleMesh
case "octahedron"   => Octahedron(center, spec.size).toTriangleMesh
case "dodecahedron" => Dodecahedron(center, spec.size).toTriangleMesh
case "icosahedron"  => Icosahedron(center, spec.size).toTriangleMesh
```

## Testing

### Unit tests — `PolytopesSuite.scala`

One suite covering all four solids. Per solid:
- Correct vertex count
- Correct triangle count
- All vertices at unit distance from origin (tolerance `1e-4f`)
- All outward face normals (dot of normal with vertex-minus-centroid > 0)
- `toTriangleMesh` returns non-empty vertex and index arrays
- `ObjectType.isTriangleMesh` returns `true` for the type name

Additional `ObjectType` tests:
- `TRIANGLE_MESH_TYPES` contains all four new names
- `TRIANGLE_MESH_TYPES` does not contain `"sphere"` or `"cube-sponge"`

### Integration tests — `scripts/integration-tests.sh`

New function `test_platonic_solids()`:
- Renders one frame of each solid with default settings
- Compares against reference images using `compare_images`
- Called from `main()` in sequential order

Reference images are generated fresh on first run after implementation and committed
alongside the code changes.

### Manual tests — `scripts/manual-test.sh`

Static tests (numbered 127–130, extending current highest of 126):
- 127: tetrahedron
- 128: octahedron
- 129: dodecahedron
- 130: icosahedron

Interactive entries in `interactive_tests` array:
- Tetrahedron interactive
- Octahedron interactive
- Dodecahedron interactive
- Icosahedron interactive

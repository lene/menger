# CUDA 13 CI Support (Task 16.5) Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add a CUDA 13 variant of the Docker image and parallel `allow_failure` CI jobs so the full test suite runs on both CUDA 12.8 and CUDA 13 in every pipeline.

**Architecture:** Parameterize the existing `optix-jni/Dockerfile` with a `CUDA_VERSION` build arg (default `12.8.0`). Build and push a second image tagged `13.2-9.0-25-1.12.0` to the GitLab registry. Add two new CI jobs (`Test:Full:Cuda13`, `Test:OptiXJni:Cuda13`) that are identical to their 12.8 counterparts but use the new image tag and have `allow_failure: true` on every rule. CUDA 12.8 remains the required gate.

**Tech Stack:** Docker, NVIDIA CUDA base images on Docker Hub, GitLab CI, sbt.

**Step 2: Add `Test:Full:Cuda13` job after `Test:Full`**

Insert the following block immediately after the closing line of `Test:Full` (after line 188, the last junit artifact line), before the `# OptiX JNI tests` comment:

```yaml
# Full test suite on CUDA 13 - informational only (allow_failure)
# Runs in parallel with Test:Full to catch CUDA version compatibility issues early
Test:Full:Cuda13:
  image: $CI_REGISTRY_IMAGE/optix-cuda:$OPTIX_DOCKER_VERSION_CUDA13
  stage: test
  needs: []
  tags:
    - nvidia  # Requires runner with NVIDIA GPU and nvidia-container-toolkit configured
  variables:
    # Request graphics capability to ensure RTX/OptiX libraries are mounted
    NVIDIA_DRIVER_CAPABILITIES: "graphics,compute,utility"
  rules:
    - if: '$CI_MERGE_REQUEST_ID && $CI_MERGE_REQUEST_SOURCE_BRANCH_NAME =~ /^hotfix\//'
      allow_failure: true
    - if: "$CI_MERGE_REQUEST_ID"
      allow_failure: true
    - if: $CI_COMMIT_BRANCH == "main"
      allow_failure: true
    - if: $CI_COMMIT_TAG
      allow_failure: true
    - if: $CI_COMMIT_BRANCH
      allow_failure: true
  before_script:
    # Verify GPU is available - fail fast if not configured
    - nvidia-smi
    # Create missing symlink for RTX core library if needed (OptiX looks for .so.1)
    - |
      if [ -f /usr/lib/x86_64-linux-gnu/libnvidia-rtcore.so.* ] && [ ! -f /usr/lib/x86_64-linux-gnu/libnvidia-rtcore.so.1 ]; then
        ln -sf /usr/lib/x86_64-linux-gnu/libnvidia-rtcore.so.* /usr/lib/x86_64-linux-gnu/libnvidia-rtcore.so.1
      fi
    # Update library cache
    - ldconfig || true
    # Install xvfb for headless LibGDX tests
    - apt-get -y update
    - apt-get -y install mesa-utils x11-xserver-utils xvfb
  script:
    # Run FULL test suite on CUDA 13
    - xvfb-run sbt test
  after_script:
    - chmod -R 777 menger-app/target/ || true
    - chmod -R 777 menger-common/target/ || true
    - chmod -R 777 optix-jni/target/ || true
  artifacts:
    when: always
    reports:
      junit:
        - menger-app/target/test-reports/**/TEST-*.xml
        - menger-common/target/test-reports/**/TEST-*.xml
        - optix-jni/target/test-reports/**/TEST-*.xml
```

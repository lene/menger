## Sprint Lifecycle

1. **Planning** - Sprint document created with goals, success criteria, and tasks
2. **Execution** - Document updated with progress, decisions, and issues
3. **Completion** - Document moved to archive with final notes
4. **Reference** - Archived document serves as historical record

## Document Format

Each sprint document typically includes:
- **Goals & Success Criteria** - What we're trying to achieve
- **Tasks & Estimates** - Detailed work breakdown
- **Technical Decisions** - Architecture and implementation choices
- **Testing Strategy** - How to verify success
- **References** - Related files and dependencies
